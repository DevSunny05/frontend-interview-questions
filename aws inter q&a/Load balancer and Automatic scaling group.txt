6.whats the diff between active and passive health checks
   1. Active Health Checks (Proactive Monitoring)
Definition
Active health checks involve the load balancer periodically sending test requests to backend instances to verify their health status.

How It Works
The load balancer sends probes (HTTP/HTTPS/TCP requests) at configured intervals.

Instances must respond within a timeout period (e.g., 200 OK for HTTP).

If an instance fails multiple consecutive checks, it's marked unhealthy.
  
  2. Passive Health Checks (Reactive Monitoring)
Definition
Passive health checks analyze real user traffic to detect failures, without sending separate probes.

How It Works
Monitors actual requests/responses:

HTTP 5xx errors

TCP connection failures

Timeouts

If an instance shows consistent failures, it's marked unhealthy

8.what is diff bet geo load server balancing and global server load balancing

   Geo Load Balancing
	What? Routes traffic based on user geographic location.

	Use Case: Localized content, compliance (e.g., GDPR).
	
	How? Uses DNS (e.g., AWS Route 53 Geoproximity Routing).

	Limitation: Slower failover (DNS-dependent).

  Global Server Load Balancing (GSLB)
	What? Smart traffic distribution worldwide based on health, latency, and policies.

	Use Case: Mission-critical apps needing low latency and instant failover.

	How? Uses anycast IPs (e.g., AWS Global Accelerator).

	Advantage: Faster failover, supports all protocols (TCP/UDP/HTTP

12.what is the purpose of target group in alb
  What is a Target Group?
A logical grouping of backend resources (EC2, Lambda, IPs) that receive traffic from an Application Load Balancer (ALB).

Key Purposes
Backend Routing

Directs traffic to specific resources (e.g., /api ‚Üí EC2, /images ‚Üí Lambda).

Health Monitoring

Regularly checks instance health (e.g., HTTP GET /health).

Automatically removes unhealthy targets.

Traffic Management

Enables sticky sessions (session affinity).

Supports multiple protocols (HTTP, HTTPS, gRPC, WebSocket).

 This is critical for stateful applications where user data is stored locally (e.g., shopping carts, user logins).
